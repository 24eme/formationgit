Gestionnaire de versions (VCS)

  • Stocker les versions des changements de fichiers
  • Parcourir cette historique de changements
  • Revenir à une version précédente

Utilisations

  • Projets informatiques
    (code source, logiciel, site web, etc...)
  • Données brutes lisibles
    (csv, config, texte, etc...)

Objectif

  • Comprendre
  • Travail en équipe
  • Revenir en arrière
  • Rassurant
  • Maintenabilité / Évolutivité

Les Outils

Centralisé

  • CVS (1990-2008)
  • SVN (2000)

Décentralisé

  • BitKeeper (1998)
  • Bazaar (2005)
  • Mercurial (2005)
  • Git (2005)

Un peu d'histoire

Créé en 2005 par Linus Torvald

Noyau linux

Remplacer BitKeeper (propiétaire)

Développé en 10 jours !

Première release en 1 an (3,5 ans pour SVN)

Succès

Tous les gros projets open source ont migré dessus :

Noyau Linux, Python, PHP, Perl, Ubuntu ...

+ de 31 millions de projets open source (en 2016)

Utilisé par 12 millions de personnes

Utilisé par Microsoft

Décentralisé

  • Ce n'est pas une architecture client / serveur
  • Tout le monde est dépôt
  • Intelligence de versionnement sur chaque dépot
  • Stockage des versions et partage séparés

Avantage de la décentralisation

  • Plus simple à déployer
  • Équipé pour gérer des conflits / fusions
  • Plusieurs modes de fonctionnement
  • Plus d'indépendance
  • Travail hors connexion
